## Таблица B. Параметры SFT (контролируемое дообучение)

Применяются ко всем 36 прогонам (4 объёма × 3 порога CoT × 3 seed).  
Подбор гиперпараметров не входит в задачу работы.  
Потери вычисляются по всем токенам последовательности. Доля промпта < 5% — Shi et al. [30].

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| Метод адаптации | LoRA | Низкоранговые адаптеры, основные веса заморожены |
| Ранг LoRA (r) | 64 | Определяет ёмкость адаптера |
| LoRA alpha | 64 | Масштабирующий коэффициент; alpha = r |
| Целевые модули | all-linear | Все линейные слои (attention + FFN) |
| Скорость обучения | 2 × 10⁻⁴ | Стандартное значение для SFT с LoRA |
| Число эпох | 3 | Полный проход по данным трижды |
| Размер пакета | 2 | На одну GPU |
| Накопление градиентов | 8 шагов | Эффективный размер пакета: 2 × 8 = 16 |
| Max length | 8 192 токенов | Верхняя граница; dynamic padding по пакету |
| Объёмы данных | 500, 1 000, 2 000, 5 000 | Вложенные подмножества (nested subsets) |
| Пороги длины CoT | 2 048, 4 096, 8 192 | Фильтрация по длине решения в токенах |
| Seeds | 42, 43, 44 | 3 прогона на конфигурацию |
| Eval temperature | 0,0 | Greedy decoding |
| Eval max_tokens | 4 096 | Аварийный потолок для обоих бенчмарков |

> **Замечание:** eval SFT-моделей использует temperature=0 и max_tokens=4096 для обоих бенчмарков. Это отличается от протокола inference-стратегий (temperature=1.0, max_tokens=2048/4096 по бенчмарку) — задокументировано как ограничение в разделе 5 ВКР.

## Таблица C. Параметры GRPO (обучение с подкреплением)

Гиперпараметры GRPO (β, learning rate) заимствованы из DeepSeekMath [16], где использовалась модель 7B.  
Масштабирование для 1,5B не исследовалось. Smoke test (100 шагов) подтвердил стабильность обучения.

| Параметр | Значение | Пояснение | Источник |
|----------|----------|-----------|----------|
| G (размер группы) | 8 | Ограничение VRAM (одна GPU A100 80 ГБ). DeepSeekMath [16] использовал G=64 | — |
| β (KL-коэффициент) | 0,04 | Штраф за отклонение от reference model | DeepSeekMath [16] |
| Скорость обучения | 5 × 10⁻⁶ | Порядок величины соответствует DeepSeekMath [16] (1 × 10⁻⁶ для 7B) | — |
| Reward | Бинарный (0/1) | 1,0 при верном ответе, 0,0 при неверном | DeepSeek-R1 [5] |
| Max completion length | 4 096 | Аварийный потолок генерации внутри GRPO | — |
| Размер пакета | 4 | На одну GPU |
| Накопление градиентов | 4 шага | Эффективный размер: 4 × 4 = 16 | — |
| Save steps | 50 | Каждый чекпоинт — отдельная точка на Парето-фронте | — |
| LoRA | r=64, alpha=64, all-linear | Та же конфигурация, что SFT | — |

> **Примечание о FLOPs:** стоимость GRPO оценивается как `C_GRPO = 10 × N × D_GRPO`, где D_GRPO — суммарное число токенов генераций из поля `num_tokens` лога TRL. Множитель k=10 является авторской оценкой верхней границы по аналогии с формулой Kaplan для SFT (k=6). Верифицированной формулы FLOPs для GRPO в рассмотренных источниках [3, 4, 16] не обнаружено.

## Таблица D. Параметры inference (стратегии инференса)

Wu et al. [3] использовали max_tokens = 1024. В настоящей работе лимиты выше, поскольку модель генерирует развёрнутые рассуждения в формате DeepSeek-R1.

| Параметр | Значение | Пояснение | Источник |
|----------|----------|-----------|----------|
| Temperature | 1,0 | Разнообразие генераций для голосования | Wu et al. [3] |
| n (генераций на задачу) | 32 | Accuracy измеряется при n = 1, 4, 8, 16, 32 | — |
| Seed генерации | 42 | Фиксирован для воспроизводимости | — |
| Max tokens (GSM8K) | 2 048 | Школьные задачи, короткие решения | — |
| Max tokens (MATH-500) | 4 096 | Олимпиадные задачи, длинные CoT | — |
| PRM scoring | log_score = Σ log(p_i) | Устойчивее prod_score на длинных решениях | Zhang et al. [14] |
| Weighted voting | Группировка + суммирование PRM scores | Формула из Math-Shepherd [13] | Wang et al. [13] |
| PRM max length | 16 384 токенов | Guard для длинных решений | — |

> **Наблюдение:** реальные T_gen при n=32 составили GSM8K ≈ 2044 токена, MATH-500 ≈ 4086 токенов — модель упирается в лимиты max_tokens. Задокументировано как ограничение в разделе 5 ВКР.
