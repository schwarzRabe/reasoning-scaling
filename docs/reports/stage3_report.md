# Этап 3 — GRPO

## Что сделано

### Smoke test (100 шагов)
- Базовая модель: c2048_v500_s42 (val_loss=0.6783)
- Adapter merge перед GRPO: W_new = W₀ + ΔW (арифметически точно)
- G=8, β=0.04, lr=5e-6, max_completion_length=4096, effective batch=16
- Reward: бинарный (1.0 верно / 0.0 неверно), компаратор is_equiv
- Чекпоинты: checkpoint-50, checkpoint-100

### Генерация
- HF generate (стандартный метод transformers), ~136 сек/шаг
- vLLM и Unsloth несовместимы с текущим стеком (TRL 1.1.0)

## Результаты

| Метрика | SFT | GRPO ckpt-50 | GRPO ckpt-100 |
|---|---|---|---|
| GSM8K acc | 0.415 | 0.415 | 0.406 |
| GSM8K none_rate | 38.4% | 36.5% | 36.5% |
| MATH-500 acc | 0.264 | 0.240 | 0.272 |
| MATH-500 none_rate | 54.8% | 55.0% | 54.2% |

- Средний reward ≈ 0.28 (28% генераций верны)
- Reward не показал выраженного роста за 100 шагов
- none_rate чуть снизился (GSM8K: 38.4% → 36.5%)
- Accuracy в пределах доверительных интервалов — модель не деградировала

## Проблемы
- 100 шагов недостаточно для значимого роста reward
- Генерация через HF generate медленная (136 сек/шаг vs ~10 сек/шаг с vLLM)
- Unsloth несовместим: конфликт TRL 1.1.0 vs требуемая ≤0.24.0
- vLLM несовместим: конфликт transformers 5.x vs 4.57.6
