# Этап 2 — SFT-обучение

## Что сделано

### Smoke test (c2048_v500_s42)
- LoRA: r=64, alpha=64, all-linear
- lr=2e-4, 3 эпохи, effective batch=16 (2×8), max_length=8192
- Валидация: 500 задач (seed=0)
- Eval: greedy (temperature=0), max_tokens=4096, парсер \boxed{}

### Полный прогон
- 36 конфигураций: 4 объёма (500, 1000, 2000, 5000) × 3 порога CoT (2048, 4096, 8192) × 3 seed (42, 43, 44)
- Статус: в процессе 
- Resume через is_run_complete: при падении Colab продолжает с места

## Результаты

**Smoke test (c2048_v500_s42):**
- train_loss: 0.5171, val_loss: 0.6783
- GSM8K: acc=0.415 [0.388, 0.442], none_rate=38.4%
- MATH-500: acc=0.264 [0.226, 0.302], none_rate=54.8%

**Наблюдения:**
- Разброс val_loss между seed < 0.001 — результат стабилен
- val_loss снижается с ростом объёма данных: 0.678 (v500) → 0.645 (v2000)
- Время обучения пропорционально объёму: 21 мин (v500) → 47 мин (v2000)

## Проблемы
- none_rate 38% (GSM8K) и 55% (MATH-500) на smoke test: модель зацикливается в <think> и не доходит до \boxed{} — ожидаемо для SFT на 500 примерах с коротким CoT
- vLLM конфликтует с transformers при совместной установке: eval отложен на отдельную сессию после обучения
- Accuracy ниже базлайна из-за смены протокола (strict parser + ChatML), не из-за деградации модели
