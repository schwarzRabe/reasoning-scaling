## Проблемы

- Cross-model PRM: обучен на Qwen-Math, генератор на DeepSeek-R1 данных — ранжирование неточно для single-selection, Best-of-N уступает Majority Voting
- Ablation с PRM, обученным на выходах нашего генератора, не проводился — выходит за рамки работы
- При temperature=1.0 accuracy при n=1 ниже, чем при greedy (0.415 vs 0.541 при temp=0.7) — ожидаемый эффект разнообразия
- vLLM KV-cache отключён для PRM вручную (патч forward) — несовместимость с forward-only scoring режимом
